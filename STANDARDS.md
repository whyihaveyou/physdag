# DAG 推导验证 — 编写与校验标准（STANDARDS）

> 2026-07-19 用户评审后确立。目的：防止"验证了，但没验证该验证的东西"这类系统性漏洞。
> 所有新推导链（`*_dag.json`）与对 `verify_dag.py` 的修改都必须遵守本文件。

## 一、两类已发生过的失效模式（血的教训）

### 模式 1：子集验证（覆盖度不足）
命题是关于任意 D / 任意参数的，check 却只验了一个特例。
- 案例：偶 D 链 E2，命题是任意偶数 D 的恒等式，check 只验了 2×2 块。
- 案例：chimera E14，check 是 `abs(0.125-0.125)<1e-9` —— 同义反复，什么都没验。

### 模式 2：验证"依据" ≠ 验证"依据的实例化"
check 验证的是被引用定理本身（或某个无关恒等式），而不是"把定理应用到
本推导的具体对象上后结论成立"。**定理为真 ≠ 代入我们的对象后结论为真**，
错误恰恰常藏在代入、换基、分量对应这些操作里。
- 案例：偶 D 链 E3，验了"任意反对称阵有块对角谱"这个定理，但没验
  "应用到 ρᵀ(γI+W/K)⁻¹(γI−W/K)⁻¹ρ 上确实得到 H(γ)=Σₖρ₂ₖ/(γ²+ωₖ²/K²)"。
- 案例：OA 链 E6，边的命题是"围道积分得 Z*=α(ω0−iΔ)"，check 却验了
  Lorentzian 归一化 ∫g=1 —— 验了个不相干的事实。

### 假 check（模式 1 的极端形态，零容忍）
以下形态一律禁止：`assert ... or True`、`assert abs(x-x)<eps`、只查 shape/类型的
"检查"、与边的命题无逻辑关系的任何 assert。
**校验器已内置静态扫描（lint）**：`or True` 直接判 FAIL，其余形态判 CONCERN。

### 变异测试（2026-07-19 起强制执行）
推荐用 `sympy_eq` 通道编写等价类 check：声明 `symbols`/`lhs`/`rhs`，
校验器机械构造 `lhs-rhs==0` 检验，并自动做变异测试——
把 rhs 取负、翻倍，要求变异后**不成立**；变异也成立的 check 是退化的
（无法区分真假结论），判 FAIL。这把"符号翻转测试"从口头自检变成每轮强制执行的机器检查。
示例见 `prx_even_dag.json` E1、`chimera_dag.json` E2。

## 二、强制规则

1. **coverage 必填**：每条非 llm 边的 check 必须标注
   `"coverage": "full" | "sampled" | "special-case"`。
   - `full`：符号证明，覆盖命题全称量词范围（任意 D、任意参数）。
   - `sampled`：数值采样（随机参数/D/初值）或有限 N/有限 T 模拟。
   - `special-case`：只覆盖了命题的一个子情形。
2. **special-case 必须写 note**：说明未覆盖的部分是什么、由 justification 的
   哪条论证兜住。校验器会在总结里单列所有 special-case 边。
3. **check 必须是实例化检验**：snippets/code 验证的对象是
   "N_i 的具体表达式 ⇒ N_{i+1} 的具体表达式"这条恒等式/残差本身，
   不是被引用的定理。推荐形态：
   - 符号：`simplify(LHS − RHS) == 0`，LHS/RHS 都取自推导原文；
   - 采样：随机参数下数值验证实例化恒等式（含扫多个 D）；
   - 残差：把解代回原始方程，在随机采样点验证残差 ≈ 0。
4. **LLM 评审补位**：`CHECKER_PROMPT` 已含反模式 5（实例化缺失）。
   justification 必须展示实例化的关键操作（代入、换基、分量对应），
   不能只甩定理名——否则评审应判 FAIL/CONCERN。

## 三、新边编写清单（Checklist）

写一条边的 check 前自问：
- [ ] 这条边的命题里有哪些全称量词（任意 D？任意反对称 W？任意解析 α）？我的 check 覆盖了吗？没覆盖→标 sampled/special-case 并写 note。
- [ ] check 验证的是不是"把依据用到本推导对象上"后的那个等式/残差？还是只验了依据本身？
- [ ] 如果我把结论改错一个符号（Z↔Z*、+↔−），这个 check 会失败吗？不会 → 是假 check，重写。
- [ ] approx 边的元数据（类型/适用条件/误差阶）齐全且误差阶记账自洽吗（相对 vs 绝对）？

第 3 条（符号翻转测试）是最有效的自检：OA 链 Z/Z* 交换 bug 就是被
"改错符号会失败"的实例化 check 抓出来的。该测试现已被 `sympy_eq` 通道
机械化（见上节"变异测试"）。

## 五、校验器机制总览（verify_dag.py）

| 机制 | 通道/字段 | 强制力 |
|------|-----------|--------|
| 覆盖度标注 | `coverage: full/sampled/special-case` | 非 llm 边必填，总结单列 special-case |
| 假 check 静态扫描 | lint（or True / abs(x-x) / shape-only） | or True 判 FAIL，其余 CONCERN |
| 机械化等价检验 | `kind: sympy_eq`（symbols/lhs/rhs） | 推荐用于等价类边 |
| 变异测试 | sympy_eq 自动执行（rhs 取负/翻倍须炸） | 变异成立判 FAIL |
| 实例化审计 | LLM prompt 反模式 5 | llm 边逐轮执行 |
| Lean 终审 | `~/chimera-lean/*.lean`，lake build 即证书 | 见下方路由规则 |

## 六、Lean 化路由规则（2026-07-19，基于三点实测成本曲线）

每条边先花 **1 分钟探库**（在 mathlib 源码 grep 关键引理名），按结果三档报价：

| 档位 | 判据 | 实测成本 | 动作 |
|------|------|----------|------|
| 有引理 | mathlib 有现成定理可直接应用 | 4-15 分钟/条 | **直接 Lean 化**（纯代数边批量做） |
| 要组装 | 引理都在但需组合/自建中间步骤 | 30-90 分钟/条 | **挑高价值边做**（分析类按需） |
| 没定理 | mathlib 有理论缺口（如块对角化归纳） | 天级以上 | **单独立项**，blueprint 先行，不混入日常验证流 |

实测数据点：E2 转置-求逆 14 min/39 行；ddim E8 模长不变 4 min/47 行（同项目有模板后边际成本递减）；OA E6 Fourier 积分 65 min/250 行（mathlib 无 Cauchy charFun、无留数定理，自建 Fourier 反演路线）。
Lean 证明是永久资产：编译通过即终身有效，不像 SymPy check 每轮重跑，也不像 LLM 评审有抽样噪声。

## 七、陈述对齐门禁（2026-07-20，借鉴 MechMath F-Reviewer）

**规则：任何 Lean 定理登记到 DAG 边之前，必须过 `lean_align.py` 对齐审查。**
形式化不是"编过就算"——专门防"为了让编译通过而弱化命题"：

1. 量词覆盖（"任意 n" 被偷改成 "n=3" 是 FAIL）；
2. 假设只减不增（新增前提是弱化；去掉前提而结论仍成立是加强，允许但注明）；
3. 结论强度（只证充分性丢必要性、不等号反向、常数/符号约定改变都是 FAIL）;
4. 对象定义对应（不允许"同名不同物"）。

工具：`~/dag-derivation/lean_align.py <dag.json> <edge|--all|--selftest>`，
单元测试 6 个合成用例（忠实/量词弱化/新增假设/符号翻转/加强/只证充分性）。
首轮实战（15 条边）即抓到 5 处真实缺口（1 真弱化 + 3 内容缺口 + 1 小引理缺失），
已由 Batch D 修复；另有 5 处属"范围批注"（边的 Lean 覆盖核心代数、
节点其余内容由前提/定义直接给出），以 `check.lean.scope_note` 注明。
审查器本身有抽样噪声（LLM 判定），FAIL 需人工复核后归类为"真弱化"或"范围批注"。
